The first task is to create the model of our form.

What is a model? Well, in general terms, a model is a representation of data or a system.

In Signal Forms, we use the 'signal' function to create a model for our form. Let's say we have a form with 'firstName', 'lastName', 'age', 'username' and 'email' fields.

First, we can create an interface that defines the structure of our form like this -

    export interface FormData {
        firstName: string;
        lastName: string;
        age: number;
        username: string;
        email: string;
    }

Now, in our component's template class, we use the 'signal' function and pass it this interface as the value of the generic type parameter -

    formModel = signal<FormData>({
        firstName: '',
        lastName: '',
        age: 0,
        username: '',
        email: ''
    });

So, we created the signal 'formModel' which will hold the form's data.

Now, we have to pass this signal to the 'form' function which is provided by ''@angular/forms/signals' package -

    loginForm = form(this.formModel);

And that's it! The 'form' function returns a new object type 'FieldTree' which represents a tree of fields in a form. If you hover over 'loginForm', you will see its type as -

    FieldTree<FormData, string | number>

To get the value of the form, we can simply use the 'value()' method on the 'loginForm' signal like this -

    this.loginForm().value()

And this will return an object with the keys as the form field names and the values as the current values of those fields. For example, it will return something like this -

    {
        firstName: 'John',
        lastName: 'Doe',
        age: 30,
        username: 'johndoe',
        email: 'john@test.com'
    }

You can get this same object if you read the 'formModel' signal as well. But ideally, when we work with forms, we should always read the value from the 'loginForm'  because it has some additional information about the form fields such as whether they are valid, whether they are dirty, etc. So, it's better to read the value from the 'loginForm' signal rather than directly from the 'formModel' signal.
